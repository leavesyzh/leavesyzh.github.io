1.RTKPOST需要的文件说明

1.观测值文件（observation data）：必须有，可以是自己接收机测得的实际数据，也可以是直接网上下载的观测站的观测值文件。

2.基准站观测值文件：单点定位不需要

3.导航电文(navigation data)：必须有。

4/5：SP3 精密轨道文件、CLK 精密钟差文件，精密星历改正需要。

6：解算结果，勾选前面的对号可以自己选择输出文件路径。

7：结算结果文件名，选好输出文件路径后会自动生成结算结果文件名，默认为.pos文件，可以修改名称。

另一部分更精细的改正，需要在options中进行参数设置。

8：卫星天线或接收机天线相位改正。

9：大地水准面模型，选择外部模型为大地水准面模型后需提供文件。

10：差分码偏差。

11：地球定位定向参数。

12：海潮负荷文件，选择潮汐改正后需提供文件。

13：电离层格网数据。

3.文件类型说明

在GNSS定位中，会使用到多种文件类型，每种文件都有特定的后缀和命名方法。以下是一些常见的GNSS文件类型及其后缀和命名规则：

1. RINEX 文件

  • RINEX 文件命名规则

    RINEX 文件完整的文件名由用于表示文件归属的 8 字符长度的主文件名和用于表示文件类型的 3 位字符长度的扩展名组成：

    $$
    _.
    $$

    • 文件名各部分释义：

      • <SITE> 为四个字符的观测站点名；
      • <RN> 为接收机的编号；
      • <CRC> 为三位 ISO 3166-1 标准的国家和地区代码，标识站点位置；
      • <S> 为数据源，即数据来源于接收机（R）还是数据流（S）；
      • <YEARDOYHRMN> 为观测开始时刻：年、年积日、时、分；
      • <LEN> 为观测时段的长度；
      • <FRQ> 为观测时的采样间隔或采样频率（星历文件无此项）；
      • <ST> 为包含的卫星系统和数据类型，第一位表示卫星系统（M、G、R、C、E、J、I）；第二位为数据类型，即观测文件（O）、导航文件（N）或气象文件（M）；
      • <FMT> 为扩展名，扩展名只有两种：rnx 或 crx。

      文件名示例：

      • ALGO00CAN_R_20170420000_01D_30S_MO.rnx 表示数据是来自加拿大的 ALGO 站 0 号接收机，于 2017 年第 42 日 0 点开始观测的，时长 1 天，采样间隔为 30 秒的多系统混合观测数据；
      • BJFS00CHN_S_20170420100_15M_01S_GO.rnx 表示数据是来自中国的 BJFS 站 0 号接收机的实时数据流，观测开始于 2017 年第 42 日 1 点，时长为 15 分钟，采样间隔 1 秒的 GPS 观测数据；
      • ALGO00CAN_R_20170420100_01H_05Z_MO.crx 表示数据是来自加拿大的 ALGO 站 0 号接收机，于 2017 年第 42 日 1 点开始观测的，时长 1 小时，采样间隔为 5 Hz 的多系统混合 Compact RINEX 观测数据；
      • ALGO00CAN_R_20170420000_01D_MN.rnx 表示数据是来自加拿大的 ALGO 站 0 号接收机，于 2017 年第 42 日 0 点开始观测，时长 1 天的多系统混合的导航数据；
      • BRDC00IGS_R_20170420000_01D_MN.rnx 表示数据来自 IGS，由 IGS 合并生成的包含多系统所有可用卫星的混合导航数据；
      • ALGO00CAN_R_20170420000_01D_RN.rnx 表示数据是来自加拿大的 ALGO 站 0 号接收机，于 2017 年第 42 日 0 点开始观测，时长 1 天的 GLONASS 系统的导航数据；
      • DAVS00ATA_R_20170420000_01D_30S_MM.rnx 表示数据来自南极洲 DAVS 站 0 号接收机，于 2017 年第 42 日 0 点开始观测，时长为 1 天的混合气象数据。

    • 参考：

      RTKLIB(一)——GNSS测量中的数据格式_gnss数据-CSDN博客

      [RINEX 3 格式简介 (gnss.help)](https://www.gnss.help/2017/04/22/rinex3-introduction/index.html#:~:text=RINEX 3 格式简介 1,观测数据文件； 2 导航（星历）文件； 3 气象数据文件。)

2. 轨道和时钟产品等文件

  • .sp3: 精密轨道和钟差文件，IGS和其他机构发布。

    IGS精密星历采用sp3格式，其存储方式为ASCII文本文件，内容包括表头信息以及文件体，文件体中每隔15 min给出1个卫星的位置，有时还给出卫星的速度。它的特点就是提供卫星精确的轨道位置。采样率为15分钟，实际解算中可以进行精密钟差的估计或内插，以提高其可使用的历元数。

    常用的sp3格式的命名规则为：

    • ttt：表示精密星历的类型，包括IGS(事后精密星历)、IGR(快速精密星历)、IGU(超快速精密星历文件)三种

    • wwww：表示GPS周；

    • d：表示星期，0表示星期日，1～6表示星期一至星期六。

    • 文件名如：igs12901．sp3，其中igs为计算单位名，1290为GPS周，1为星期一。

    • 参考：RTKLIB(一)——GNSS测量中的数据格式_gnss数据-CSDN博客

  • .clk: 精密钟差文件，包含卫星钟差和钟漂的时钟校正数据。

  • .erp: 地球自转参数文件，包含极移和世界时数据，这些参数可以影响地球表面点的准确定位和时间标准的同步。

    1. 极移（Polar Motion）: 描述地球旋转轴相对于地球表面的移动。
    2. 世界时（UT1-UTC）: 描述世界时与协调世界时（UTC）之间的差异。
    3. 日长（Length of Day, LOD）: 描述地球自转速率的变化

  • .ion: 电离层文件，提供电离层总电子含量数据。

  • .atx：天线文件。

    GPS从入门到放弃（二十） — 天线偏移 | 航行学园 (voycn.com)

  • .gcc：用于存储GNSS协调系统之间的转换参数，包含了坐标转换参数、误差模型以及其他与不同坐标系统之间转换相关的信息。

  • .dcb：用于存储GNSS卫星和接收机的码偏差数据。码偏差是由不同频率的信号在接收机和卫星设备中经历不同的延迟引起的。主要用途包括：

    1. 校正GNSS观测数据: 减少由于码偏差导致的误差，提升定位精度。
    2. 电离层研究: 提供准确的码偏差信息，支持电离层总电子含量（TEC）的计算和研究。
    3. 多频GNSS数据处理: 提供不同频率信号间的偏差校正，提高多频定位解算的准确性。

    以 GPS 双频 C1-P2 数据为例，进行无电离层组合时需加入 P1-C1 DCB 值将基准修正到 GPS 无电离层组合的钟差基准。

    通过引入一定的基准，可将差分码偏差还原为绝对码偏差。用户进行处理时，则只需要根据选择的观测值类型选择相应的绝对码偏差，如同卫星钟差和接收机钟差修正一样，不涉及到其他类型的码偏差参数，因而也不存在组合问题。

  • .bia：多系统码偏差和相位偏差校正文件。

  • .obx：包含卫星的姿态四元数

3. 气象和电离层文件
  • .met: 气象数据文件，包含温度、气压、湿度等数据。
  • .ionex: 电离层网格文件，提供电离层总电子含量的网格化数据。
4. 辅助文件
  • .log: 日志文件，记录接收机的操作和观测过程。
  • .sum: 总结文件，提供观测数据的摘要信息。
  • .snx: SINEX（Solution Independent Exchange Format）文件，包含站点坐标和速率等数据，用于GNSS解算结果的交换。
